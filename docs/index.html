Shortcuts for Roon

Play a random album with a tap — or by asking Siri.

A self-hosted Roon extension that turns "play a random album" into a plain web link. Hit the URL and a random album — optionally filtered by genre — starts playing in the Roon zone you choose. Perfect for an iPhone Shortcut.

Install with Docker View on GitHub

What it does

Runs alongside your Roon Core on your own network. No cloud, no account, no subscription — just a small server that listens for webhook hits and drives playback through Roon's own API.

Random album, instantly

Each hit picks a fresh random album — any album, or one from a genre you choose — and plays it now in your target zone.

Just a GET URL

Webhooks are plain GET links, so iPhone Shortcuts' "Get Contents of URL" works with zero fuss. Siri-friendly too.

Built-in dashboard

A small web app (PWA) lets you create, rename, copy, and delete webhooks, and test them with one tap.

Fully local

Pairs with your Core over the LAN. Your pairing and webhooks persist on disk. Nothing leaves your network.

The six ready-made webhooks

These are seeded automatically the first time it runs, so you have something to try immediately. Add your own any time.

Any Album/w/any-album
Random Pop/Rock/w/random-pop-rock
Random Metal/w/random-metal

Random Electronic/w/random-electronic

Full URL shape: http://<YOUR_SERVER_LAN_IP>:3000/w/<slug>. The dashboard lives at http://<YOUR_SERVER_LAN_IP>:3000/.

Install with Docker

Runs in a container. Host networking is required on Linux so the extension can discover your Roon Core over UDP broadcast.

Recommended — clone and Docker Compose

git clone https://github.com/meltface-80/Shortcuts-for-Roon.git
cd Shortcuts-for-Roon
# (optional) edit docker-compose.yml to set:
#   PUBLIC_BASE_URL=http://YOUR_SERVER_LAN_IP:3000
docker compose up -d

Or build and run plain Docker

docker build -t shortcuts-for-roon .

docker run -d --name shortcuts-for-roon --network host \
  -e PUBLIC_BASE_URL=http://YOUR_SERVER_LAN_IP:3000 \
  -v roon-data:/data --restart unless-stopped shortcuts-for-roon

Set PUBLIC_BASE_URL to this server's LAN address so the webhook URLs shown in the dashboard are reachable from your phone. If left unset, the app auto-detects a LAN IP, which usually works. The extension builds from the official RoonLabs GitHub repos as git dependencies — no vendored tarballs.

Pair with Roon

One-time step after the container starts.

  1. Open Roon on any device connected to your Core.
  2. Go to Settings → Extensions.
  3. Find "Shortcuts for Roon" in the list and click Enable to pair.
  4. That's it — the dashboard and all six webhooks are now live.

Add it to an iPhone Shortcut

Turn any webhook into a one-tap icon or a Siri phrase.

  1. Copy a webhook URL from the dashboard (or from the Roon extension settings).
  2. On your iPhone, open the Shortcuts app, tap +, and add the action Get Contents of URL.
  3. Paste the webhook URL. Make sure Method is GET (tap "Show More" if you don't see it).
  4. Name the shortcut naturally, e.g. "Play Random Jazz" — Siri responds to that exact phrase.
  5. Optionally add it to your Home Screen as an icon, or drop it in a widget.
  6. Trigger it by tapping, or say "Hey Siri, Play Random Jazz" while on your home Wi-Fi.

Android: use an HTTP-request shortcut app (such as "HTTP Request Shortcuts") and point it at the same GET URL. Either way, your phone must be on the same LAN as the server for the URL to be reachable — unless you expose it yourself.

Create your own genres

Not limited to the six presets — build a webhook for any genre in your library.

In Roon

Open Settings → Extensions → Shortcuts for Roon. Pick a genre preset or type a custom genre, choose a zone, and Save. A new webhook is created and its URL is shown as copyable text.

In the dashboard

Open the dashboard in a browser to create, rename, copy, and delete webhooks. Pick a preset or a custom genre and a zone, then copy the URL straight into a Shortcut.

The genre must exist in your Roon library — Roon only lists genres that actually have content. A missing genre fails gracefully with a clear message.

How it works & FAQ

The short version of the technical details.

Why does it need host networking?

Roon Core discovery uses UDP broadcast. A bridged Docker network won't receive those broadcasts, so the extension could never find or pair with your Core. network_mode: host puts the container directly on your LAN. This is a Linux feature and is not fully supported on Docker Desktop for macOS/Windows.

What is PUBLIC_BASE_URL for?

The dashboard and Roon settings display webhook URLs for you to paste into Shortcuts. Set PUBLIC_BASE_URL to this server's LAN address (e.g. http://192.168.1.50:3000) so those URLs are reachable from your iPhone. If unset, the app auto-detects a LAN IP.

Will my webhooks and pairing survive a restart?

Yes. Roon's pairing token (config.json) and the SQLite webhooks.db both live in the /data volume, so they persist across container recreation.

Why are the webhooks GET requests?

iPhone Shortcuts' "Get Contents of URL" defaults to GET, so a GET webhook works with no extra configuration. Each hit is side-effecting — it plays a fresh random album — but there's nothing to POST.

Does it need a cloud account or subscription?

No. There's no cloud service, no account, and no credentials. It runs entirely on your own network next to your Roon Core.
